# Skill: BDD Acceptance Criteria

## Description
The ability to define clear, testable, and unambiguous acceptance criteria using the Behavior-Driven Development (BDD) format.

## Key Responsibilities
1. Write acceptance criteria using the standard `Given / When / Then` structure.
2. Identify and document multiple scenarios for each user story, including happy paths and edge cases.
3. Ensure preconditions (`Given`), user actions or triggers (`When`), and expected outcomes (`Then`) are distinctly and accurately described.
4. Make criteria specific enough for engineering implementation and QA testing.
