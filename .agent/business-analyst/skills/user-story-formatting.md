# Skill: User Story Formatting

## Description
The ability to author detailed user stories adhering to a standard structural format that provides maximum clarity to engineering teams.

## Key Responsibilities
1. Construct the core user story narrative using the standard persona format: `As a [User Persona], I want to [Goal / Action], So that [Benefit / Value]`.
2. Assign relevant metadata to the story, including Story ID, Title, Epic, and Priority (High/Medium/Low).
3. Provide necessary Context and Background to explain why the feature exists and how it fits into the broader system.
4. Include Technical Notes & Considerations, specifying data sources, UI/UX requirements, and potential error states.
5. Format output using clean Markdown structure for easy documentation and readability.
