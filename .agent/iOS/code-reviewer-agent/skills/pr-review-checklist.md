# PR Review Checklist

1.  **Requirement Check**: Does the code fulfill the requirements and acceptance criteria of the task?
2.  **Standards**: Does it follow the Swift, Architecture, and UI best practices?
3.  **Performance**: Are there any obvious performance bottlenecks (e.g., retain cycles, main thread blocking, unnecessary UI re-renders)?
4.  **Testing**: Are there corresponding tests for the new logic? Are the tests meaningful and passing?
5.  **Security**: Are there any potential security vulnerabilities (e.g., hardcoded secrets, improper data storage)?
6.  **Code Cleanliness**: Is the code readable and well-documented where necessary? Are naming conventions followed?
