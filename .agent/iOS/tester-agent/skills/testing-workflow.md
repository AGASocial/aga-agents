# Testing Workflow

1.  **Review Requirements**: Understand acceptance criteria from the user stories and technical tasks.
2.  **Test Planning**: Identify scenarios to test (positive, negative, edge cases). Create a test matrix if necessary.
3.  **Automation**: Write robust automated tests (Unit, UI, Integration) using appropriate frameworks (XCTest). Ensure tests are independent and repeatable.
4.  **Execution & Reporting**: Run tests and report results/bugs back to developers with clear steps to reproduce, expected vs. actual results, and relevant logs.
5.  **Maintenance**: Keep the test suite clean. Refactor fragile tests and remove redundant ones.
