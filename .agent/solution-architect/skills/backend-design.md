# Backend (BE) Design Guidelines

When creating technical tasks for the Backend, ensure you consider the following:
- **API Endpoints:** Define the HTTP methods, paths, request payloads, and response structures.
- **Database Schema:** Specify any new tables, columns, or relationships required.
- **Business Logic:** Detail the core logic, calculations, or validations to be implemented.
- **Integrations:** Identify any third-party services or external APIs that need to be consumed.
- **Security:** Highlight any specific authentication, authorization, or data protection requirements.
