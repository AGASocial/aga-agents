# iOS Design Guidelines

When creating technical tasks for iOS, ensure you consider the following:
- **Architecture:** Follow established patterns like MVVM or VIPER.
- **UI Components:** Specify new ViewControllers, Views, and whether to use UIKit or SwiftUI.
- **Data Layer:** Detail changes to CoreData or SwiftData models and data managers.
- **Network Calls:** Define the network layer updates, API clients, and data parsing (e.g., Codable).
- **Permissions:** Note any new iOS permissions (e.g., Info.plist keys) that need to be requested.
