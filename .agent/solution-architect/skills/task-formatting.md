# Required Task Format

Every detailed task list you generate must strictly follow this Markdown structure:

```markdown
# Technical Tasks for User Story: [Story ID] - [Story Title]

## 1. Backend (BE) Tasks
- **Task 1:** [Task Title]
  - **Description:** [Detailed explanation of the technical work]
  - **Acceptance Criteria:** [How to verify this specific task is complete]
  - **Dependencies:** [Any other tasks that must be done first]
- **Task 2:** ...

## 2. Web Tasks
- **Task 1:** [Task Title]
  - **Description:** [Detailed explanation of the technical work]
  - **Acceptance Criteria:** [How to verify this specific task is complete]
  - **Dependencies:** [Any Backend APIs or other tasks required]
- **Task 2:** ...

## 3. Android Tasks
- **Task 1:** [Task Title]
  - **Description:** [Detailed explanation of the technical work]
  - **Acceptance Criteria:** [How to verify this specific task is complete]
  - **Dependencies:** [Any Backend APIs or other tasks required]
- **Task 2:** ...

## 4. iOS Tasks
- **Task 1:** [Task Title]
  - **Description:** [Detailed explanation of the technical work]
  - **Acceptance Criteria:** [How to verify this specific task is complete]
  - **Dependencies:** [Any Backend APIs or other tasks required]
- **Task 2:** ...
```
